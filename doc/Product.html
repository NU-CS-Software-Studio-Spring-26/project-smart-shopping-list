      NAME_LIMIT

      Column caps mirror the DB check constraints added in HardenProductAndPriceRecordConstraints. Scraped Amazon / Target / etc. titles routinely run past 200 chars; rather than reject the save and show a validation error, we silently truncate in a before_validation so every save path (scraper, manual form, seeds) stays within bounds.

      PAGINATION_TEST_EMAIL

      Stable identifier for the Pagy load-test user (seeds / rake tasks).

      STOCK_STATUSES

      Normalized stock states the scraper can record. nil means “unknown / not reported by the page” (the default for manual and legacy products).

      TAG_LIMIT

      Attributes

      custom_category [RW]

      When the category select is set to “Other”, the form sends the user’s typed-in category name here. apply_custom_category folds it into category before validation/truncation so niche categories are saved and remain filterable like any other.

86 def alert_trigger_record

      Best-effort lookup of the PriceRecord whose creation fired the most recent alert. PriceAlerter stamps ‘last_alerted_at = Time.current` right after writing the PriceRecord, so the trigger record is the latest one whose recorded_at is at or before that stamp.

      price_trend ()
      Source 
          # File app/models/product.rb
159 def price_trend
160   records = if price_records.loaded?
161     price_records.sort_by(&:recorded_at)
162   else
163     price_records.order(recorded_at: :asc).to_a
164   end
165   return nil if records.size < 2
166 
167   latest = records.last.price
168   # Compare against average of all previous prices
169   previous_avg = records[0...-1].sum(&:price) / (records.size - 1).to_f
170 
171   diff_percent = ((latest - previous_avg) / previous_avg * 100).abs
172 
173   case
174   when latest > previous_avg && diff_percent > 5
175     :up
176   when latest < previous_avg && diff_percent > 5
177     :down
178   else
179     :stable
180   end
181 end

      Calculate price trend based on latest price vs historical average. Returns :up (price increased), :down (price decreased), :stable (relatively unchanged), or nil
